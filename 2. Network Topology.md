## Objective - Create a logically isolated virtual network to host studio assets.
In the initial stage of the Ideal Studios migration, focus was placed on network secuirty and logical isolation. Rather than using a default VPC architecture, a custom environment was architected to ensure full control over data flow and access.

---

### VPC Configuration
* VPC name: Ideal-Studions (IS) 
* CIDR block: 10.0.0.0/16
* Tenancy: Default
* Region: us-east-1

---

### Subnet Segmentation
To implement a robust **3-Tier architecture**, the network was partitioned into specific zones
#### 1. Public subnet (Web Tier)
* IPv4 CIDR: 10.0.1.0/24
* Function: Hosts the IS-Server(EC2) to handle ingress traffic from cloudfront and end users.

#### 2. Private subnet (Data Tier)
* IPv4 CIDR: 10.0.2.0/24
* Function: Hosts the database (is-db). This subnet is isolated with no direct route to the internet gateway making it inaccessible by the public.

---

### Routing and Connectivity
1. Intenet Gateway: This was created and attached to the VPC.
2. Public route table: Destination set to **0.0.0.0/0** with the Target as the IGW, **igw-**
3. Association: Linked ot the public subnet to enable web access.
