# Objective -To eliminate latency for international clients and establish an entry point for Ideal Studios.
This phase focuses on Global Content Delivery and Granular Security, ensuring the studio's assets are fast for clients and safe from internal accidental changes.

### 1. Amazon CloudFront (The CDN)
While Ideal-Studios servers are in Virginia (us-east-1),the clients are global. I implemented Amazon CloudFront to cache 3D renders at Edge Locations worldwide.
* Benefit: This offloads 90% of traffic from the Studio's backend servers to AWS's global network, ensuring zero-latency viewing of high-resolution studio renders.
* Security at the Edge: Enforced SSL/TLS Termination with a "Redirect HTTP to HTTPS" policy, ensuring all user data is encrypted in transit.

### 2. Route 53 DNS Management
To make the studio accessible via a readable URL `idealstudios.com.ng`, I configured a Route 53 Hosted Zone.
* AWS Alias Records: Used Alias A-Records instead of standard CNAMEs. This allows for faster DNS resolution and seamless integration with the CloudFront distribution.
* Scalability: The architecture is now "Domain-Ready," allowing us to easily add subdomains like `portal.idealstudios.com.ng` for future client dashboards.
