## Objective - Establish a functional web presence for Ideal-Studios

### The Server Set-up
  The aim here is to create a web presence for the firm.
* Instance Type: `Amazon Linux 2023 EC2`
* Subnet: `Public`
* Automation: Utilised bash script to automate the installation of Apache(HTTPD) and the deployment of a Bootstrap landing page.
* Security: Restricted `SSH` access strictly to my specific IP address.
    * Implemented a stateful firewall via `Secuirty Groups`
    * Managed Linux file permissions using `chmod 400` to secure the `.pem` private key.
---
### The Workstation
 To support the studio's workflow, I configured the the EC2 instance as a secure workstation for the architects to  upload their blueprints.
 It was implemented as follows:
*  Create a dedicated directory for blueprints
	  `mkdir -p /home/ec2-user/blueprints`
*  Set permissions so the 'ec2-user' can write to it
  	`chown ec2-user:ec2-user /home/ec2-user/blueprints`
* Create a test CAD file to show it works
    `touch /home/ec2-user/blueprints/mall-project-v1.dwg`
---
### How it works
* Each architect would have a unique Linux identity ( e.g `arch_Mark`, `arch_john`)
* Instead of passwords, architects would use Public/Private key pairs. The public key would be pasted into the `~/.ssh/authorized_keys` file to grant access.
* When an architect at Ideal Studios opens an app like `FileZilla` or `WinSCP` on their computer, they can drag and drop their large CAD files from their PC directly into the `~/blueprints` folder.
* If an architect leaves the firm, deleting the public key instantly revokes access without affecting others.
